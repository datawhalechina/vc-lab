# 第1章：认知唤醒与问题洞察

**核心目标**：理解设计的多元价值，掌握双钻模型，完成从“感觉”到“课题”的转化。

本章聚焦双钻模型的第一个钻石：先广泛发现问题，再收敛定义核心问题。方向错误，努力白费。定义问题，就是寻找那个最值得投入的、正确的起点。

## 1.1 连接：设计的三种视角

设计远不止于外观美化。它是连接人与物的桥梁，体现在三个层面：

1. **解决问题**：像修缮房屋，系统化地平衡功能、结构、技术，让产品不仅“能用”，更要“好用”。
2. **创造体验**：像烹饪菜肴，关注用户与产品交互时的情绪、感受与认知，塑造完整的体验关系。
3. **探索未来**：像绘制航海图，基于技术与社会趋势，探索未来人与技术、环境的新关系。

### 探索任务

请拿起手边最常用的一个物件，例如水杯、耳机或键盘，并记录在你的探索日志中：

- 用“修缮房屋”的视角，思考它解决了什么**问题**。
- 用“烹饪菜肴”的视角，描述它带来的**体验**，包括触感、视觉和感受。
- 用“绘制海图”的视角，想象它未来 5 年可能如何**演变**。

## 1.2 思维导航仪：双钻模型

面对复杂问题，我们需要可靠的“思维导航仪”：**双钻模型**。它强调两次“发散-收敛”的循环：

- **第一个钻石（问题空间）**：**做正确的事**。先广泛发现问题，再聚焦定义核心问题。
- **第二个钻石（方案空间）**：**把事情做正确**。为已定义的问题探索多种方案，并交付最佳方案。

## 1.3 从“感觉”到“课题”

### 步骤一：绘制痛点地图

聚焦你的创作、工作或学习场景，用下面的格式快速记录所有“不爽”时刻：

```text
[人]在[时间]下的[场景]中，当需要做[某件事]时，我需要[X步操作]，这打断了我，因为[Y原因]。
```

目标：至少记录 5 个具体痛点。

**产出**：一张痛点地图。可以使用便利贴、思维导图或白板。

### 步骤二：明确核心目标

从痛点地图中，选出 1-2 个最困扰、最值得解决的核心痛点，为其填写课题定义书。

**产出**：1 份核心痛点的课题定义书。

### 工具卡 1：课题定义书

这个表格帮助你将模糊的“不爽”转化为清晰、可执行的设计课题。

| 项目 | 填写内容 |
| --- | --- |
| 目标用户 | 谁在这个场景中遇到问题？ |
| 场景与时机 | 问题发生在什么时间、地点和流程中？ |
| 具体痛点 | 用户具体卡在哪里？发生了什么不便？ |
| 影响后果 | 这个痛点造成了效率、体验、情绪或成本上的什么损失？ |
| 现有替代方式 | 用户现在如何绕过或临时解决这个问题？ |
| 设计机会 | 如果问题被解决，用户会得到什么关键价值？ |
| 初步课题 | 用一句话写出你准备深入探索的设计课题。 |

### 阶段回顾与心态调整（一）

完成课题定义书后，请暂停片刻，进行第一次阶段性回顾。

1. **回顾与检视**：回顾你从“模糊感觉”到“清晰课题”的过程。哪个洞察最意外？这个课题是否真正触及了核心不便？当前思考是否存在盲点？
2. **心态与展望**：认可自己发现问题、定义问题的努力。这是创造的宝贵起点。保持好奇，期待接下来的探索。

## 1.4 侦察兵任务：理解战场格局

在明确自身课题后，需要系统性地“侦察战场格局”：进行竞品分析，理解现有解决方案的优劣，发现价值空地。

请围绕你的核心课题，建立至少 5 份竞争对手档案。

**产出**：一份包含至少 5 个竞争对手档案的侦察报告。

### 工具卡 2：竞争对手档案分析框架

这个框架帮助你系统性地理解市场现状，发现设计机会。

| 维度 | 记录内容 |
| --- | --- |
| 竞品名称 | 产品、服务、工具或替代方案名称 |
| 目标用户 | 它主要服务谁？ |
| 核心功能 | 它如何解决相似问题？ |
| 使用流程 | 用户完成关键任务需要经历哪些步骤？ |
| 优势 | 哪些设计值得学习？ |
| 不足 | 哪些痛点仍未被解决？ |
| 差异机会 | 你的方向可以在哪些方面形成独特价值？ |

## 1.5 绘制战略地图：从发散到聚焦

通过前期的痛点发散、选项确认与竞品分析，我们已经积累了大量信息。此时，信息是发散的，需要从多个可能方向中，聚焦到一到两个最值得深入设计的核心问题。

问题收敛常用两种工具：

1. **HMW 提问法（How Might We，我们可以怎样……）**：将模糊痛点转化为可切入的设计机会。通过“我们可以怎样……”的提问方式，从多个角度重新表述问题，为后续方案探索打开多条路径。
2. **POV 框架（Point of View，观点陈述）**：明确定义“目标用户、需求与核心洞察”。格式为“作为一名【用户角色】，我想要【需求/目标】，因为【洞察/原因】”。POV 能够将问题锁定在具体用户与情境中，避免设计方向过于宽泛。

本阶段的核心产出是：一个明确的 POV 陈述和 3 个以上不同角度的 HMW 问题。

### 学习者任务

产出你的“战略地图”，包含：

1. 选定的核心收敛方向。
2. 至少 3 个不同角度的 HMW 问题。
3. 一句明确的 POV 使命宣言。

### 示例

**HMW 角度**：

- 效率角度：HMW 让用户在创作过程中减少因工具切换造成的注意力损失？
- 学习角度：HMW 让新用户快速掌握键盘的 AI 协同功能？
- 情感角度：HMW 让键盘成为用户桌面上的“创作伙伴”而非“冷工具”？

**POV**：

> 作为一名高强度内容创作者，我想要键盘能预判我下一步的创作动作，因为每次手动调用工具都会打断我的心流状态。

### 阶段回顾与心态调整（二）

完成战略地图后，进行第二次阶段性回顾。

1. **回顾与检视**：POV 是否清晰定义了“为谁”和“为何”？HMW 问题是否从不同侧面打开了解决问题的可能性？基于竞品分析，你的方向是否具有独特性或优势？
2. **心态与展望**：为你发现的市场空白或用户未被满足的深层需求感到鼓舞。这是一个创造独特价值的潜在机会。保持专注，准备进入具体的概念塑造阶段。
