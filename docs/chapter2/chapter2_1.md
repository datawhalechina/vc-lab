# 第2.1节：绘制产品设计蓝图

**核心目标**：将抽象的“问题-用户-需求”转化为具体的产品概念与可视化设计方向。

本节将第 1 章产出的战略地图（POV + HMW）转化为一份可执行的产品设计蓝图。

## 工作坊：绘制你的设计蓝图

设计蓝图不是最终方案，而是概念成形前的“共同语言”。它帮助你把问题、用户、价值、功能、交互和气质放到同一张图中，判断产品概念是否连贯。

**产出**：一份完整的产品设计蓝图。

## 工具卡 3：产品设计蓝图绘制指南

从下面几个维度具象化你的产品概念。

![产品设计蓝图绘制指南](/course/product-design-blueprint.png)

| 模块 | 关键问题 | 产出 |
| --- | --- | --- |
| 用户与场景 | 产品主要服务谁？发生在什么具体场景？ | 目标用户、关键场景 |
| 核心问题 | 它回应第 1 章中的哪个 POV？ | 问题陈述 |
| 价值主张 | 用户为什么需要它？它带来什么新价值？ | 一句话价值主张 |
| 核心功能 | 为解决问题，产品必须具备哪些能力？ | 3-5 个核心功能 |
| 交互方式 | 用户如何触发、操作、反馈和完成任务？ | 关键交互流程 |
| 形态方向 | 产品可能是什么形态？硬件、软件、模块或服务？ | 形态草图或描述 |
| 设计语言 | 它应该呈现什么气质？冷静、温润、专业、未来感？ | 3-5 个语义词 |
| 验证方式 | 如何快速判断这个概念是否被理解？ | 访谈、纸模、流程演示或视觉原型 |

## 工具卡 4：语义定义与转化

语义定义帮助你将抽象气质转化为可沟通、可执行的设计语言。

### 语义定义的三个层次

| 层次 | 说明 | 示例 |
| --- | --- | --- |
| 核心语义 | 产品最重要的气质，决定整体方向 | 专注、可信、灵动 |
| 支撑语义 | 辅助表达核心语义的关键词 | 克制、轻盈、亲和 |
| 设计属性 | 可被设计执行的外观、材料、交互和声音属性 | 低饱和色、细腻纹理、短路径反馈 |

### 语义-属性关联矩阵

| 语义词 | 造型 | 色彩 | 材料/质感 | 交互/反馈 |
| --- | --- | --- | --- | --- |
| 温润 | 圆角过渡、柔和曲线 | 低饱和暖色 | 木质、磨砂、细腻表面 | 反馈缓和、不突兀 |
| 克制 | 简洁比例、减少装饰 | 黑白灰或中性色 | 金属、玻璃、哑光材质 | 操作路径短、提示明确 |
| 未来感 | 几何切面、模块化结构 | 冷色或高对比点缀 | 半透明、发光、复合材料 | 即时反馈、状态可视化 |

## 多元视角验证（一）

完成产品设计蓝图后，邀请课程群中至少 3 位背景不同的伙伴，也可以小队内互相帮助，向他们展示蓝图并收集反馈。

单一视角容易产生盲点，多元视角能帮助我们发现隐藏的问题，激发新的灵感，让设计基础更加稳固。请记录他们的主要观点，并思考如何整合这些不同视角，优化设计方向。
